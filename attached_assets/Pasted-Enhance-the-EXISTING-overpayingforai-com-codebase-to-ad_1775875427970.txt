Enhance the EXISTING overpayingforai.com codebase to add a lightweight affiliate system, centralized affiliate resolver, and reusable CTA injection across the site.

Do NOT rewrite the app.
Do NOT change branding.
Do NOT remove existing calculator, decision engine, pricing refresh, AI types, comparison pages, guides, or best pages.
Extend current architecture only.

GOAL
Create robust monetization plumbing so recommendation sections, winner sections, best-pick sections, and affiliate cards can all resolve to:
1) affiliate URL when available
2) internal fallback URL when affiliate is missing
3) optional disabled/non-primary state when neither is available

This system must support the real-world admin workflow where affiliate programs are still being registered and some providers are pending.

==================================================
CORE REQUIREMENTS
==================================================

Build a centralized affiliate config and resolution layer.

Create or update these pieces:

1) Central affiliate data source
Suggested file:
- src/data/affiliates.json
or
- src/data/affiliates.ts

Each provider/tool entry should support:
- id
- name
- affiliateUrl (nullable)
- directUrl (optional canonical product URL if used in your current system)
- fallbackUrl (internal route like /compare/... or /decision-engine)
- status: active | pending | unavailable
- ctaLabelPrimary (optional)
- ctaLabelSecondary (optional)
- notes (optional)
- updatedAt (optional)

Example shape:
{
  "cursor": {
    "id": "cursor",
    "name": "Cursor",
    "affiliateUrl": null,
    "fallbackUrl": "/compare/cursor-vs-copilot",
    "status": "pending",
    "ctaLabelPrimary": "Start with Cursor",
    "ctaLabelSecondary": "Compare alternatives"
  }
}

2) Affiliate resolver utility
Suggested file:
- src/utils/affiliateResolver.ts

Implement reusable logic that:
- resolves the best outbound/internal target for a given tool/provider
- prefers affiliateUrl when present
- otherwise falls back to fallbackUrl
- returns metadata needed for rendering CTA state

Suggested exported functions:
- getAffiliateTarget(toolId)
- hasAffiliateLink(toolId)
- getPrimaryCta(toolId, context?)
- getSecondaryCta(toolId, context?)

Return object should support:
- href
- isExternal
- isAffiliate
- status
- label
- fallbackUsed
- rel/noreferrer rules when external

3) Reusable CTA component(s)
Suggested files:
- src/components/monetization/AffiliateCta.tsx
- src/components/monetization/RecommendationCtas.tsx

Build components that:
- render primary CTA
- optionally render secondary CTA
- handle external vs internal links cleanly
- preserve design system
- support context-specific labels
- degrade gracefully if affiliate link is missing

4) CTA injection across site
Patch recommendation-heavy pages/components so they use the centralized resolver rather than page-specific ad hoc links.

Targets likely include:
- compare pages
- best pages
- AI type detail pages
- recommendation cards
- winner sections
- cheapest sections
- top-pick modules
- end-of-page CTA sections
- any “Start with X” / “Try X” / “Use X” buttons

==================================================
CTA BEHAVIOR RULES
==================================================

Primary logic:
- If affiliate exists:
  use affiliate link as primary CTA
- Else if fallback exists:
  use fallback as primary CTA
- Else:
  use a safe internal default such as decision engine or calculator

Secondary logic:
- If affiliate is primary:
  secondary CTA should go to internal comparison/details page when available
- If fallback is primary:
  secondary CTA can go to pricing/calculator/decision engine

Default CTA label behavior:
- affiliate active:
  “Start with {Tool}”
- pending affiliate:
  “Compare {Tool}”
  or
  “See cheaper alternatives”
- cheapest/winner contexts can override label:
  “Try the cheapest option”
  “See pricing breakdown”
  “Compare alternatives”

Do NOT leave dead buttons.
Do NOT leave empty hrefs.
Do NOT hardcode random URLs throughout pages.
