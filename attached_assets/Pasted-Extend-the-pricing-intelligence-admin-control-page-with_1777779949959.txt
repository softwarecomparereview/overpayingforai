PART 4 — HOW AUTOPILOT WORKS EXPLAINER

Add a collapsible section on /admin/pricing-intelligence-control titled:

"How the pricing autopilot works"

Content must be clear enough for a non-engineer founder.

Include these sections:

1. What it checks
- trusted pricing sources
- official provider pricing pages
- docs
- source registry

2. What it extracts
- vendor
- tool/model
- pricing or plan change
- usage-limit change
- hidden-cost change
- confidence
- source URL

3. How it routes changes
- AUTO: safe, high-confidence, trusted-source change
- REVIEW: plausible but needs human review
- ALERT: high-impact, major, suspicious, or commercially important change
- REJECTED_LOW_CONFIDENCE: too weak to publish

4. What "manual no-update" means
- checks sources
- classifies possible changes
- shows proposed updates
- does not update public site
- does not change tracker/history
- safe for review

5. What "dry run" means
- can simulate pipeline output
- may write preview or audit files
- does not publish public updates

6. What "full run" means
- runs live pipeline
- writes safe AUTO items
- queues REVIEW/ALERT items
- updates run log
- may update tracker/history depending on route

7. What gets published automatically
- only high-confidence trusted-source AUTO items
- never low-confidence items
- never review/alert items

8. What still needs human review
- major price changes
- hidden costs
- source conflicts
- ambiguous wording
- non-primary sources
- suspicious or commercially sensitive changes

9. Safety rules
- append-only history
- no overwriting history
- no automatic publishing for low confidence
- no direct commits to main
- all runs logged
- source shown for every change
