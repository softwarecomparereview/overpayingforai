PART 1 — FIX CALCULATOR CREDIBILITY

Find the calculator/recommendation logic.

Fix these trust-breaking bugs:
- Free plans must not automatically win just because price is $0.
- Free tiers must be excluded or downgraded when the user’s usage exceeds realistic free-tier limits.
- Writing tools like Rytr must never be recommended for API-heavy/model-token users.
- Tool recommendations must match user intent/category.

Add or enforce these fields in tool/plan data if missing:
- category: chat | api | writing | coding | image | productivity
- supportsApiUsage: boolean
- supportsTokenUsage: boolean
- freeTierLimitDescription
- minViableUsageThreshold
- recommendedUseCases

Recommendation rules:
- If user scenario is API/token-heavy, only consider API-capable/model providers.
- If user scenario is writing/marketing, writing tools may be considered.
- If plan price is 0, it can be shown as “free option” but cannot be the primary paid alternative unless usage is within the free-tier threshold.
- If free tier is insufficient, show: “Free tier likely insufficient for this usage.”
- Never show “100% savings” as the main result unless it is genuinely valid and usage-compatible.

Add tests or scenario checks for:
- low usage individual
- high usage individual
- 50M token API user
- freelancer
- marketer
- developer
- agency/team
- budget user
- undecided buyer
- ChatGPT Plus user comparing alternatives

Output should include:
- recommendation
- rationale
- category match reason
- savings estimate if credible
- warning if estimate is not apples-to-apples
